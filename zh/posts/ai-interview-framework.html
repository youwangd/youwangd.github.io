2026-03-19 面试 职业 AI

面试还在考反转链表？醒醒，2026年了

Leetcode考的是AI已经比人类做得更好的技能。一个2026年版软件工程师面试框架——考判断力、评审能力、架构思维，而不是背算法。

你们还在考这个？

2026年一个senior engineer的日常是什么样的？review AI生成的代码，设计多智能体协作的系统架构，debug模型幻觉产生的逻辑错误，决定哪些模块可以交给AI写、哪些必须人类把关。

然后走进面试间，面试官打开一道medium难度的leetcode："请实现一个LRU Cache。"

我们在为一个已经不存在的岗位面试。

标准的软件工程师面试——刷题、系统设计白板、行为面试STAR法则——是为一个工程师主要产出是手写代码的时代设计的。但那个时代正在消失。Copilot补全了60%的按键。Claude Code一行prompt写出完整feature。Cursor的background agent在你开standup的时候就把refactor做了。

面试没变，工作变了。这就像招一个摄影师，考试内容是暗房冲洗胶片。

行业开始定义新标准

2026年初，一件有意思的事情正在发生：认证体系开始重新定义「合格工程师」的含义。

Anthropic 在3月12日发布了「Claude Certified Architect, Foundations」——第一个来自前沿AI实验室的架构师认证。闭卷考试，60道题，120分钟，不能用Claude也不能查文档。前置条件：至少6个月的Claude API + MCP生产环境实战经验。认证有效期只有6个月——因为这个领域变化太快了。背后是Anthropic一亿美金的Partner Network。

AWS 今年1月推出了「Agentic AI Demonstrated」微认证和配套课程，重点是用Bedrock agents、Amazon Q和LangChain构建自主AI系统。新的「Generative AI Developer – Professional」认证(AIP-C01)考RAG pipeline、Bedrock agents、knowledge bases、guardrails、向量存储。同时老的ML Specialty认证(MLS-C01)在3月31日退役。

Microsoft 将在6月30日退役Azure AI Engineer Associate (AI-102)，替换为新的AI原生认证：SQL AI Developer (DP-800)和Cloud & AI Security Engineer (SC-500)。Google 2月发布了AI Professional Certificate，并更新了Professional ML Engineer考试，加入GenAI和Vertex AI内容。

每一家主要云厂商都在做同一件事：用考智能体系统、prompt engineering、RAG架构、生产环境AI部署的新认证，替换考传统ML pipeline的旧认证。

认证体系在往前跑。面试流程还在原地站着。

能力变了，你变了吗

工程师的核心工作已经从写代码变成编排写代码的系统。现在真正重要的能力是：

判断力——知道该做什么、不该做什么、哪些可以交给AI。这是2026年区分senior和junior的关键能力，跟动态规划没有半点关系。

评审能力——AI生成的代码看起来都很合理。能不能看出隐藏的bug？安全漏洞？藏在整洁variable name后面的O(n³)？AI写的代码是「看起来对」的代码。工程师的价值在于知道「看起来对」什么时候不等于「真的对」。

架构思维——AI可以实现，但AI不会做系统分解、判断failure mode、决定边界在哪里。抽象层越高，越需要人。

表达能力——prompting就是跟另一种听众沟通。能清晰表达意图的工程师——不管对象是人还是机器——产出翻倍。想不清楚的人写不出好prompt，也写不出好需求文档。

大厂 vs. 创业公司：谁先动

不是所有公司都在同一个速度上。

Canva 现在要求工程候选人在技术面试中使用AI工具——Copilot、Cursor、Claude。他们的态度很明确：面试应该反映工程师每天的真实工作方式。非工程岗位也鼓励在面试全流程中使用AI。

Google 走了相反的方向。他们在恢复线下面试来防止AI作弊，并推出了新的预筛工具Google Hiring Assessment。重点不是重新设计面试内容，而是让旧流程更难被hack。

这个分化完全可以预料。大公司不改是因为leetcode可以规模化。培训200个面试官评价leetcode很容易，培训200个面试官评价「判断力」很难。改面试流程意味着重新培训面试官、更新rubric、重建calibration体系——在FAANG规模下，这是一个巨大的组织工程。

国内大厂可能更慢。字节、阿里、腾讯的面试流程已经高度标准化，刷题文化根深蒂固。笔试+多轮技术面+HR面的模板用了十年，候选人的准备方式也完全围绕这个模板。改变面试等于改变整个招聘漏斗，牵一发而动全身。

创业公司和中型公司（Canva、Anthropic生态伙伴、AI原生团队）可以快速变革——面试官团队小、流程灵活、工程团队本身就在daily使用AI工具。

最讽刺的是：做AI产品的公司，面试的时候不让用AI。

一个实用框架

那到底该考什么？以下是5轮面试框架，每轮针对AI时代真正重要的能力：

第一轮：AI辅助开发（90分钟）

给候选人一个真实问题、一台装了Claude Code / Cursor的电脑、90分钟时间。评价的不只是最终代码，而是过程。他是先拆解问题再prompt，还是上来就让AI生成？他验证了输出吗？他发现了AI遗漏的edge case吗？他知道什么时候该停止prompt、自己写吗？

考察：工具熟练度、判断力、问题拆解能力、验证直觉。

第二轮：AI代码评审（60分钟）

展示三份AI生成的代码提交。一份有微妙的逻辑错误（edge case处理不当）。一份有安全漏洞。一份代码正确但架构很差。候选人review这三份代码，写出feedback，解释理由。

考察：评审能力、安全意识、架构品味、表达清晰度。

第三轮：含AI约束的系统设计（60分钟）

标准系统设计，但加一个twist：系统的一半将由AI智能体实现。候选人需要决定哪些组件委托给AI、哪些必须人类实现、如何在生产环境验证AI生成的组件。加入成本约束——token budget是真实的。

考察：架构思维、设计中的AI意识、成本推理、生产思维。

第四轮：在AI写的烂代码里找bug（60分钟）

把候选人扔进一个500-1000行的代码库，全是AI生成的，看起来挺像那么回事。里面埋了三个bug：一个逻辑错误，一个性能问题，一个只在并发场景下炸。可以用任何工具。关键不是能不能修好——而是怎么找到它们的。

考察：debug思路、读陌生代码的能力、在不确定的情况下系统性地排查问题。

第五轮：技术决策（45分钟）

场景题：「团队需要重写一个核心服务。有人提议用AI生成80%的代码，一周搞定。另一个方案是三个工程师花六周。你怎么决定？」追问trade-off：可靠性、可维护性、团队知识沉淀、bus factor、测试策略。

考察：判断力、技术领导力思维、风险评估、trade-off表达能力。

丢掉什么

注意这个框架里没有什么：

Leetcode算法题。 AI解这些比人快也比人好。考这个就像计算器时代考心算。

八股文。「进程和线程的区别是什么？」如果需要知道，三秒钟查到。

禁止用AI的take-home项目。 不可能执行，而且测的是错误的东西。让候选人用任何想用的工具，然后评价结果和过程。

面试官自己也慌了

这个框架有一个尴尬的地方：它更难评价。Leetcode有标准答案。「他是不是会用AI？」这事儿没有标准答案。

所以面试官自己得是每天用AI干活的人。你自己都不用Claude Code，怎么评价别人用得好不好？面试官的门槛提高了，不是降低了。

大公司坚持用leetcode不是因为它有效，是因为它好操作。培训200个面试官评价leetcode很容易。培训200个面试官评价判断力？难。

但好操作和有用是两回事。

往哪走

认证体系走在了面试流程前面。AWS、Anthropic、Google、Microsoft都重新定义了2026年工程师该会什么——智能体系统、RAG pipeline、生产环境部署、架构判断力。不是背算法。

面试永远是个proxy。说白了就是在猜：「这个人来了能不能干活？」这个猜法得跟实际工作对上。

2020年的工作：写代码、review代码、设计系统、跟人沟通。

2026年的工作：指挥AI写代码、review AI生成的代码、设计包含AI组件的系统、跟人和机器沟通。

能力变了。面试也该变了。

不然就继续考反转链表吧。反正AI正在替我们写未来。
